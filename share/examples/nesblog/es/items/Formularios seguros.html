Formularios seguros

En que se basa la seguridad de los formularios Nes


Si fuésemos a crear un robot para enviar post con estupendas ofertas de todo tipo de productos milagrosos, o incluso con buenas intenciones, lo primero que deberíamos hacer es visitar la página del formulario y ver y guardar los campos, para luego hacer algo así:

http://nes.sourceforge.net/miniblog/es/sign.html?nombre=elnombre&comentario=elcomentario&enviar=enviar

Esto con nuestro sistema no funcionará nunca.

Puedes crear tu formulario como lo harías con cualquier otro, con tus nombres de campos normales para que el código de tu aplicación sea legible, posteriormente mediante unos tags de nes {: & obfuscated nombre_del_campo :} puedes ofuscar los campos de manera que nunca tengan el mismo nombre, cada vez que se cargue la página, el nombre de los campos cambiará.

Opcionalmente se le puede añadir un catpcha.

Plugin: Forms

Plugin: Captcha
